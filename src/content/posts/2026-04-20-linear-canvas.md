Linear 這次把 Figma 式的無限畫布帶進了自己的產品裡，正式進入公開測試。這不是 Linear 第一次做白板類功能——兩年前他們也試過一次，但當時只是在 issue 上貼個簡單的 sticky note。這次完全不同。

## Canvas 是什麼？

一張無限畫布，但每個元素都是 Linear 原生的物件——issue、project、roadmap 項目、或 Markdown 文件。你可以把它們拖到畫布上、用連線表達關係、用 frame 分組，而這些物件和 Linear 主系統是雙向同步的。

換句話說：**你在 Canvas 上拖動一張 issue card，對應的 issue 狀態真的會更新**。

## 實測兩週後的感想

我們在一個 6 人的產品小組上試用兩週，分工是 2 位 PM、2 位設計師、2 位工程師。結論可以分成兩面：

**取代得了的**：
- Notion 裡的大部分 PRD（尤其是需要和 roadmap 連結的）
- Miro / FigJam 上那種「便利貼 + 關係連線」的 planning 會議
- 過去散落在多個工具裡的 quarterly planning

**取代不了的**：
- Figma 裡純設計探索的 moodboard
- 需要複雜白板工具的 workshop（有票投、timer 這類功能的）

## 誰該導入？誰該觀望？

如果你的團隊已經在用 Linear 當主要 project management 工具，這是個不需要思考就該開啟的功能。如果你現在還在用 Jira 或 Asana——Canvas 本身不會讓你想換過去，但它可能是一個訊號：**工具整合的時代正在取代工具組合的時代**。
